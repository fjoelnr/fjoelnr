# Fjölnir Óðalriksson

I build local-first AI systems, Home Assistant tooling, Linux hardware fixes, and embedded automation that has to survive real hardware, real networks, and real operational drift.

## What I Work On

- AI-native repository workflows and agent operating structure
- Home Assistant integrations, review-first automation flows, and local LLM tooling
- Linux hardware enablement on actual devices instead of generic lab targets
- ESP32 and MQTT systems for sensing, monitoring, and autonomous edge behavior

## Public Repositories

### AI-Native Tooling

- [`ai-native-repo-template`](https://github.com/fjoelnr/ai-native-repo-template): reusable template for AI-native repositories with stack profiles, guardrails, and maintainer context
- [`anr-cli`](https://github.com/fjoelnr/anr-cli): CLI for bootstrapping, migrating, and validating ANR repositories
- [`minecraft-llm-agents`](https://github.com/fjoelnr/minecraft-llm-agents): local-first multi-agent experimentation in Minecraft

## Working Principles

- reproducible first
- local-first when privacy, latency, or operational control matter
- review before write
- automation with explicit boundaries, backups, and dry-runs

## Current Direction

The portfolio is being consolidated around a smaller set of public repositories with clearer scope, stronger documentation, and visible operational discipline. The goal is not maximum repo count. The goal is a smaller number of repos that are defensible, useful, and actually maintained.
